## Critical correction: split adjustment

Yahoo's chart API does **not** retroactively split-adjust many small-cap/OTC
series. Example: GDC's 1:200 reverse split (2026-06-27) made $0.014 bars sit
next to $2.80 bars, fabricating a "+11,795%" winner that single-handedly
carried whole configurations. `YahooMarketData` now restates pre-split bars
on the post-split basis using Yahoo's split events; all bars were re-synced.

**Impact: v1's "+3.6% avg 5d vs −1.4% control" was largely this artifact.**
On corrected data the ungated strategy loses money. This is exactly why the
audit trail exists.

## What identifies the huge winners

Winner-vs-loser feature medians (baseline run, 153 winners / 3,102 losers):

| Feature | Winners | Losers |
|---|---|---|
| Entry price | **$4.30** | $33.89 |
| Volume z-score (signal day) | **+1.21** | +0.09 |
| Pre-entry 3-day run-up | **+14.4%** | +0.8% |
| Dollar volume | $44M | $178M |

Walk-forward logistic regression on the full candidate set (6,570 events,
train ≤ 2026-05-13, test after — fully out-of-sample):

- **precision@25 = 16.0% vs 4.3% base rate (3.8x lift)**; precision@100 = 14% (3.3x).
- Coefficients (standardized): `log_dollar_volume −0.27` (smaller stocks win),
  `pre_return_3d +0.13` (momentum helps), `zscore +0.11` (mention acceleration
  helps), `volume_z +0.09`, `breadth −0.16` (!), `sentiment +0.01` (~nothing).

Three uncomfortable, load-bearing findings:

1. **Momentum-exclusion kills the signal** (pre-run ≤ 10% gate: 0.5x lift).
   Combined with `pre_return_3d`'s positive weight, the hypothesis "forum buzz
   fires *before* the move" is not supported — buzz + early move + volume
   confirms *continuation*. The system is a momentum-confirmation engine, not
   a crystal ball. That can still be tradable, but it must be sold as what it is.
2. **Lexicon sentiment contributes nothing** (weight ≈ 0). The FinBERT/LLM
   sidecar (Phase 2 leftover) is now justified by data, not hope.
3. **Author breadth is negatively predictive** in this window — concentrated
   posting (fewer authors per mention) precedes big moves more often than
   organic breadth. Consistent with pumps being... effective, short-term. The
   live engine's +20% breadth weight is likely wrong; refit before trusting it.

## Tradability verdict (decision gate status: NOT passed, one live candidate)

The best gated configuration (price ≤ $5 + volume z ≥ 2) is `net ≈ breakeven`
at 5% friction with a 22% chance of a ≥+30% spike within a week — i.e. the
edge is real but the naive "buy and hold 5 days" exit gives it all back on
the losers (median −5.9%). The obvious next lever is exit design (stop-loss /
take-profit asymmetry), which requires intraday assumptions we should treat
carefully, plus live forward-testing.
